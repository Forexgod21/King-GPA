# outputs/

## Purpose

The `outputs/` folder holds generated deliverables and prepared final artifacts for King-GPA assignments.

Outputs are where the actual work produced by the workflow lives: drafts on their way to completion, structured support files, packaged final files, and validation-ready handoff artifacts. It is the delivery side of the system, separate from doctrine, pipelines, skills, and templates.

---

## What Outputs Cover

- **Drafts** — In-progress deliverables that are not yet finalized
- **Structured support files** — Organized working files that support a deliverable
- **Packaged final files** — Complete deliverables ready for handoff
- **Validation-ready handoff artifacts** — Files that have passed QA and are ready to submit

---

## How Outputs Are Organized

- Outputs should stay organized by assignment or example
- Each assignment gets its own space so files do not bleed across deliverables
- Outputs should not be confused with proof of external software execution unless proof artifacts are actually present
- Outputs should preserve the distinction between generated work and verified work — a generated draft and a manually verified, screenshot-backed final file are not the same thing, and the folder structure should make that visible

---

## Rule

Outputs must reflect the real completion state of the assignment.
