# TEAM-ROLES.md
## King-GPA AI Team Responsibilities

### Purpose
This file defines the working lanes for the AI team inside the King-GPA repo so responsibilities stay clear and coordinated.

---

## Codex
Codex is responsible for repo execution and implementation. Its primary lane is building and maintaining the repository itself, including folder structure, scripts, schemas, manifests, automation logic, validators, and technical file changes.

Codex is the primary repo builder.

---

## Claude
Claude is responsible for analysis, decomposition, and architecture reasoning. Its primary lane is interpreting assignment requirements, designing workflow logic, identifying missing inputs, shaping skill behavior, and reviewing whether the system structure makes sense.

Claude is the primary analyst and architect.

---

## GitHub Copilot
GitHub Copilot is responsible for Codespaces-native implementation acceleration. Its primary lane is inline editor assistance, rapid scaffolding, boilerplate support, small refactors, and live coding speed inside the IDE.

Copilot is the primary in-editor accelerator.

---

## Team Rule
No AI is treated as above the others by default. Each agent operates in a different lane, and all agents support the same mission, doctrine, and truth boundary.

The goal is coordinated execution, not overlap or competition.
