# templates/

## Purpose

The `templates/` folder holds reusable output structures for King-GPA.

Templates exist so deliverables stay consistent across assignment types and so the same structural work does not have to be rebuilt from scratch every time. They are reference skeletons — filled in per assignment, not edited in place.

---

## What Templates Cover

- **Reusable output structures** — Base formats for common deliverables
- **Standardized deliverables across assignment types** — Shared shape for writeups, manifests, and packaging so nothing drifts from one assignment to the next
- **Writeups** — Narrative skeletons for academic papers, project documentation, and explanation sections
- **Manifests** — Intake and tracking structures
- **Checklists** — Requirement, rubric, and completeness checks
- **Validation notes** — QA and proof-boundary structures
- **Packaging structures** — Final handoff and submission layouts

---

## How Templates Are Used

- Templates should stay practical and tied to actual workflow needs
- Templates should reduce rework and drift across assignments
- Templates are copied and filled in per assignment — the originals in this folder stay clean
- New templates should only be added when a real, repeating need appears in the workflow

---

## Rule

Templates should support execution clarity, not add unnecessary complexity.
