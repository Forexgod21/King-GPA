# templates/database_assignment_intake_checklist.md

## Purpose

This file is a reusable intake checklist for academic database assignments. It is copied per assignment and filled in during intake — not edited in place.

Its job is to identify what exists, what is missing, and what must still happen outside the repo before execution begins.

---

## Assignment Understanding Checks

- [ ] Assignment instructions reviewed
- [ ] Rubric reviewed if available
- [ ] Assignment type confirmed as database
- [ ] Required deliverables identified
- [ ] Part 1 / Part 2 split identified if applicable

---

## Input Collection Checks

- [ ] Source files collected
- [ ] Existing draft collected if available
- [ ] Topic or scenario identified
- [ ] Physical model collected if it already exists
- [ ] Screenshots or proof artifacts collected if they already exist

---

## Model Readiness Checks

- [ ] Tables identified
- [ ] Fields identified
- [ ] Primary keys identified
- [ ] Foreign keys identified where needed
- [ ] Data types planned or confirmed
- [ ] Missing model flagged if not yet available

---

## Software Boundary Checks

- [ ] Determine whether Microsoft Access or another external database tool is required
- [ ] Determine whether actual build proof must exist
- [ ] Determine whether screenshots are required
- [ ] Keep planning separate from real software execution

---

## Rule

Database intake must identify what exists, what is missing, and what must still happen outside the repo before execution begins.
