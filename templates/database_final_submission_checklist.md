# templates/database_final_submission_checklist.md

## Purpose

This file is a reusable checklist for academic database assignments before final handoff or submission. It is copied per assignment and filled in with real status — not edited in place.

Its job is to confirm that structure, writing, and proof artifacts all match the real completion state before anything is treated as done.

---

## Assignment File Checks

- [ ] Assignment instructions present
- [ ] Rubric present if available
- [ ] Manifest present
- [ ] Assignment type identified

---

## Database Structure Checks

- [ ] Physical model exists
- [ ] Tables are defined
- [ ] Primary keys are defined
- [ ] Foreign keys are defined where needed
- [ ] Data types are planned
- [ ] Build sheet exists if Access execution is required

---

## Proof Artifact Checks

- [ ] Screenshots exist only if actually captured
- [ ] Access build is only marked complete if actually done
- [ ] Proof artifacts are attached only if they exist

---

## Writeup Checks

- [ ] Part 1 and Part 2 stay separated if required
- [ ] Writeup matches the actual model/build state
- [ ] Placeholders are removed
- [ ] Execution claims match reality

---

## Final Package Checks

- [ ] Required sections are present
- [ ] File names match assignment requirements
- [ ] Final files are organized cleanly
- [ ] Package is ready for handoff or submission

---

## Rule

A database assignment is not submission-ready until structure, writing, and proof artifacts all match the real completion state.
