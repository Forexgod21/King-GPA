# skills/

## Purpose

The `skills/` folder holds assignment-type support modules for King-GPA.

Skills are not general-purpose tools. Each skill exists to support a specific class of academic IT work and provides the structured logic needed to handle that work consistently.

---

## What a Skill Is

- A skill is an **assignment-type support module**
- Each skill exists to support a **specific class of academic IT work**
- Skills are scoped narrowly so the workflow stays predictable

---

## What Each Skill Should Eventually Define

Every skill should grow to define:

- **Supported task type** — What kind of assignment this skill handles
- **Required inputs** — What materials must be present before execution can begin
- **Execution logic** — How the work is actually carried out
- **Validation logic** — How outputs are checked against requirements
- **Common output patterns** — Typical deliverables produced for this assignment type

Skills should stay practical and tied to actual workflow needs. They are not theoretical templates — they exist because the work demands them.
