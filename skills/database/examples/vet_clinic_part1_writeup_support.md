# skills/database/examples/vet_clinic_part1_writeup_support.md

## Purpose

This file supports the written Part 1 section of the vet clinic database assignment. Part 1 is the conceptual explanation: it defines core database terms, explains what a relational database is, introduces Microsoft Access objects, and walks through the standard database design method.

Part 1 is about concepts and understanding. It is not about build execution, screenshots, or proof of completion — those belong to Part 2.

This file guides what Part 1 should cover. It does not write the student's paper.

---

## What Part 1 Should Explain

Part 1 should clearly cover:

- **Core terms** — The meaning of database, table, record, field, and value
- **Hierarchy** — How those terms relate: a database contains tables, tables contain records, records contain fields, and fields hold values
- **Relational databases** — What a relational database is and how tables are linked through key relationships
- **Microsoft Access database objects** — The six objects: Tables, Queries, Forms, Reports, Macros, and Modules
- **Five-step database design method** — The standard process: determine the purpose of the database, determine the tables needed, determine the fields needed, identify fields with unique values (primary keys), and determine the relationships between tables
- **Connection to the vet clinic example** — Where useful, tie the concepts back to the vet clinic scenario (e.g., the Clients table holds records for each pet owner, each record has fields like FirstName and LastName, and ClientID is the primary key that links to the Pets table)

---

## Suggested Structure

A simple paragraph-form outline the student can follow:

1. **Opening paragraph** — State the purpose of Part 1: to explain the core database concepts that underpin the vet clinic database design.

2. **Core terms paragraph** — Define database, table, record, field, and value in plain language, and explain the hierarchy between them.

3. **Relational database paragraph** — Explain what makes a database "relational," how tables are linked through shared keys, and why this structure matters for organizing related information like clients and their pets.

4. **Access objects paragraph** — Introduce the six Microsoft Access database objects (Tables, Queries, Forms, Reports, Macros, Modules) and briefly describe the role of each.

5. **Design method paragraph** — Walk through the five-step database design method and, where helpful, connect each step to the vet clinic example to show how the method was applied to produce the physical model.

6. **Closing paragraph** — Summarize how these concepts prepare the reader to understand the vet clinic database design that will be built in Part 2.
