# qa/

## Purpose

The `qa/` folder holds validation notes, requirement checks, completeness checks, and proof-boundary checks for King-GPA assignments.

QA is not about style opinions or generic feedback. It exists to confirm that an assignment actually meets its stated requirements, that nothing is missing, and that claimed proof artifacts really exist before anything is treated as submission-ready.

---

## What QA Files Cover

- **Validation notes** — Record of what was checked and what was found
- **Requirement checks** — Line-by-line confirmation that the assignment rubric or instructions were met
- **Completeness checks** — Confirmation that every required deliverable is present
- **Proof-boundary checks** — Confirmation that screenshots, exported files, and other evidence actually exist and were not assumed, described, or simulated

---

## How QA Is Used

- QA files should help determine whether work is ready for handoff or still missing required elements
- QA should stay tied to the actual assignment requirements, not generic opinions
- QA should help distinguish generated work from verified work
- QA should flag blockers clearly instead of hiding them

When a QA check fails, the failure is surfaced — not smoothed over. A missing screenshot, an unfilled placeholder, or a skipped rubric item becomes a visible blocker, not a footnote.

---

## Rule

QA exists to protect truth, completeness, and submission readiness.
